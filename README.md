## TRPG 回放

点击主界面右上角的“🎬 回放”打开回放工作室。DM 可以选择公开、指定队伍、指定玩家或 GM 范围，开始实时录制，也可以从当前战役的既有聊天生成回放。历史台词会按中英文字符和标点估算阅读时长，单句最长 9.75 秒，并以约 0.27 秒的短间隔连续播放。默认使用 15 FPS 快速导出（约为 30 FPS 一半的截图数量），也可选择 12、24、30 或 60 FPS；点击“渲染并导出 MP4”后，应用会隐藏编辑器界面、逐帧渲染场景与台词层，并通过 PATH 中的 FFmpeg 输出 H.264 MP4。DM 的角色立绘和姓名显示在左侧，玩家显示在右侧；每个 QQ 发送者使用稳定且不同的专业配色，角色图片会优先复用 QQ 消息图片缓存，保持原始比例并显示在姓名牌上方。

配置 `.env` 中的 `DEEPSEEK_API_KEY` 后，可以在回放工作室开启“允许 DeepSeek 润色台词并控制镜头”。“自定义导演要求”输入框可指定角色口癖、节奏和镜头风格，最多发送 2000 字。导演使用开启思考模式的 `deepseek-v4-pro`，返回逐句润色台词以及受限的构图和运动指令；本地会验证每个原始台词索引恰好出现一次，再重新计算台词时长并生成平滑镜头轨迹。DeepSeek 只接收所选发布范围已经允许显示的台词和角色模型是否存在，不得新增剧情事实或扩大可见范围。“录制 DM 自由镜头”默认关闭，只有手动开启后才持续采集 DM 镜头。

“DeepSeek 导演并导出”提供一键输出：重建可见聊天、等待 DeepSeek 返回润色台词和逐句构图/运动方案、本地验证后将镜头强制对准正在发言且拥有场景立牌的角色；找不到对应角色时才按方案生成极慢的短距离环境移动，然后逐帧渲染并编码 MP4。环绕运动已禁用，镜头转换会覆盖约 70% 的当前台词并限制每句的物理移动距离，避免跨越飞船的快速运镜。DeepSeek 通过受限镜头指令控制成片节奏，不直接执行任意坐标或访问隐藏内容。

DeepSeek 完成后可在“查看 DeepSeek API 原始响应”中查看并复制 API 返回的 `message.content`；API 密钥和模型思考过程不会显示。“DeepSeek 导演并导出”会等待有效导演方案真正返回并应用后再开始渲染，不再先生成本地环绕镜头。视频默认使用 Joth 的 CC0 曲目《JRPG2 Piano》，这是一首可循环的经典 JRPG 风格钢琴音乐，可在导出区关闭或调整音量。原始 MP3 和来源/CC0 许可说明保存在 `assets/audio`，FFmpeg 会按视频长度循环、淡入淡出并编码为 AAC。

视频配音使用本地 Spark-TTS 中文模型，不上传聊天内容。首次使用前运行 `powershell -ExecutionPolicy Bypass -File scripts/setup_sparktts.ps1`，脚本会通过项目约定的本机代理安装官方运行环境和模型，并在 `.data/willowblossom/tts/spark` 生成 32 个固定中文角色音色（16 个男声、16 个女声）。应用通过单个常驻 GPU 进程加载模型，每个 QQ 号会稳定分配一个音色，也可在带滚动条的角色语音设置中手动选择或随机更换；每句台词都使用所选固定参考音色进行克隆，因此不会因文字长短而改变角色声线。DM/玩家声像在导出视频中分别偏左/偏右。DeepSeek 导演同时返回正常显示的 `text` 和只供语音使用的 `speech_text`，把 AI、英文缩写和阿拉伯数字改成中文谐音或读法；本地规则会在缺失时兜底，导出画面仍显示原始正常文字。播放与 MP4 导出从同一时间线起点使用完全相同的镜头、字幕、语音文本和角色设置。整体语速默认 1.30 倍，可无上限统一调整；整体台词停留默认 1.00 倍，也可无上限统一缩放字幕停留、台词间隔和导演镜头时间。两项设置均随回放项目 JSON 保存。

公开回放只包含公开消息；队伍和玩家回放在写入事件前执行访问检查。GM 回放可能含私聊和隐藏内容，界面会显示发布警告。停止播放或完成视频渲染后会恢复 DM 播放前的体素场景。可选的 `*.willow-replay.json` 只用于保存和再次编辑回放项目，不是最终交付格式。
