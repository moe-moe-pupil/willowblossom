## TRPG 回放

点击主界面右上角的“🎬 回放”打开回放工作室。DM 可以选择公开、指定队伍、指定玩家或 GM 范围，开始实时录制，也可以从当前战役的既有聊天生成回放。历史台词会按中英文字符和标点估算阅读时长，单句最长 9.75 秒，并以约 0.27 秒的短间隔连续播放。默认使用 15 FPS 快速导出（约为 30 FPS 一半的截图数量），也可选择 12、24、30 或 60 FPS；点击“渲染并导出 MP4”后，应用会隐藏编辑器界面、逐帧渲染场景与台词层，并通过 PATH 中的 FFmpeg 输出 H.264 MP4。DM 的角色立绘和姓名显示在左侧，玩家显示在右侧；每个 QQ 发送者使用稳定且不同的专业配色，角色图片会优先复用 QQ 消息图片缓存，保持原始比例并显示在姓名牌上方。

配置 `.env` 中的 `DEEPSEEK_API_KEY` 后，可以在回放工作室点击“整理现有台词”。“自定义整理要求”输入框可用来指定提要格式、关注重点、术语和措辞，最多发送 2000 字；重新整理同一版台词会用新要求覆盖旧提要。制作提要使用开启思考模式的 `deepseek-v4-pro`；DeepSeek 只接收所选发布范围已经允许显示的台词，自定义要求不能扩大可见范围、续写剧情、改写玩家发言或自动决定镜头。视频镜头使用 DM 实时录制的自由镜头轨迹，从历史聊天生成时则使用当前镜头，确保导出结果可复现。

“DeepSeek 整理 + 自动导演并导出”提供一键输出：重建可见聊天、请求 DeepSeek 制作提要、本地导演优先对准正在发言角色的场景立牌；找不到对应角色时生成可复现的缓慢漂移镜头，然后立即逐帧渲染并编码 MP4。为保持故事权和访问边界，DeepSeek 本身不能执行摄像机或导出工具；镜头和导出由本地确定性导演执行。

DeepSeek 完成后可在“查看 DeepSeek API 原始响应”中查看并复制 API 返回的 `message.content`；API 密钥和模型思考过程不会显示。视频默认加入本地程序化生成的原创舒缓纯音乐，可在导出区关闭或调整音量。音乐按视频长度生成柔和和弦、琶音、立体声变化和淡入淡出，再由 FFmpeg 编码为 AAC，因此不需要下载音乐或附带第三方授权。

视频配音默认使用本地 Sherpa-ONNX Kokoro INT8 中英文模型，不上传聊天内容；模型放在 `.data/willowblossom/tts/kokoro-int8-multi-lang-v1_1`，以 24 kHz 输出并提供 100 个中文角色音色（3–57 女声、58–102 男声），模型缺失时回退到 Windows TTS。播放预览会在后台预生成并缓存整段回放的语音；如果某句尚未完成，时间轴会短暂停在台词边界而不会静默跳过，暂停、跳转或下一句出现时当前语音同步停止。每个 QQ 号会稳定分配不同的 ONNX 说话人，DM/玩家声像在导出视频中分别偏左/偏右。“角色语音设置”窗口可为每个角色手动选择 Kokoro 音色并调整基础语速和相对音量；Windows 声音与音调作为回退设置。这些设置随回放项目 JSON 保存，同时用于预览与导出。逐帧渲染期间保持静音，全部画面完成后再一次性生成语音并与原创音乐混入 MP4。自动语速限制在 0.85–1.45 倍，并且导出不再通过改变采样速度强制压缩长台词，避免含混和变调。送入模型前会启用本地中文词典、在中英文边界自动加入停顿，并将模型不认识的表情符号替换为短停顿；画面上的原始台词不变。导出区可以关闭预览与导出语音或调整总音量。历史台词显示时间和切换间隔在上一版基础上加倍（最短 2.7 秒、最长 9.75 秒、间隔约 0.27 秒）。

公开回放只包含公开消息；队伍和玩家回放在写入事件前执行访问检查。GM 回放可能含私聊和隐藏内容，界面会显示发布警告。停止播放或完成视频渲染后会恢复 DM 播放前的体素场景。可选的 `*.willow-replay.json` 只用于保存和再次编辑回放项目，不是最终交付格式。
